# Runbook Templates

These files are templates only. They define the fields an operator should fill during a real operation, drill, incident, rotation, handover, complaint, or restore event.

Do not fill these templates with fake incidents, fake outage evidence, fake alert delivery proof, fake rotation records, fake restore records, or placeholder operational artifacts. If no real event or drill has occurred, leave the template unused or keep a private draft clearly marked as not evidence.

Before committing any completed record, review it against `docs/evidence/redaction-policy.md` and `SECURITY.md`. Do not commit secrets, raw private logs, private backup paths, DB URLs with passwords, webhook URLs, TLS/ACME private material, private operator artifacts, or unredacted correspondence.
